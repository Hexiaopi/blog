## 代理模式(Proxy Pattern)
> 由于某些原因需要给某对象提供一个代理以控制对该对象的访问。这时，访问对象不适合或者不能直接引用目标对象，代理对象作为访问对象和目标对象之间的中介。

### **角色**

- 抽象主题：通过接口或抽象类声明真实主题和代理对象实现的业务方法
- 真实主题：实现了抽象主题中的具体业务，是代理对象所代表的真实对象，是最终要引用的对象
- 代理类：提供了与真实主题相同的接口，其内部包含有对真实主题的引用，它可以访问、控制或扩展真实主题的功能。
